# Azure Container Instances

## Objective
Deploy and manage a containerized application using Azure Container Instances (ACI) without managing virtual machines or Kubernetes infrastructure.

## Services Used
- Azure Container Instances (ACI)
- Docker Container Image
- Azure Container Registry (Optional)
- Azure Portal
- Resource Group

## Prerequisites
- Active Azure Subscription
- Azure Portal Access
- Basic understanding of Containers and Docker

## Steps
1. Create a Resource Group.
2. Open Azure Container Instances in the Azure Portal.
3. Create a new Container Instance.
4. Select a container image from Docker Hub or Azure Container Registry.
5. Configure CPU and memory requirements.
6. Assign a DNS name label and networking settings.
7. Review the configuration and deploy the Container Instance.
8. Verify that the container is running successfully.
9. Access the application using the public IP address or DNS name.
10. Stop or delete the Container Instance after testing.

## Outcome
Successfully deployed and managed a containerized application using Azure Container Instances, providing a fast, scalable, and serverless solution for running containers without managing underlying infrastructure.
