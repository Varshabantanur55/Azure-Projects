# Azure Monitor with Alerts

## Objective
Monitor the health and performance of Azure resources using Azure Monitor and configure alerts to receive notifications when specific conditions or thresholds are met.

## Services Used
- Azure Monitor
- Azure Virtual Machine
- Alert Rules
- Action Group
- Azure Portal
- Resource Group

## Prerequisites
- Active Azure Subscription
- Azure Portal Access
- Basic understanding of Azure Monitoring

## Steps
1. Create a Resource Group.
2. Deploy a Windows Virtual Machine.
3. Open Azure Monitor from the Azure Portal.
4. Create an Action Group for alert notifications.
5. Create an Alert Rule for the Virtual Machine.
6. Configure the alert condition (such as CPU Percentage greater than 80%).
7. Associate the Action Group with the Alert Rule.
8. Review and create the alert.
9. Generate activity to trigger the alert.
10. Verify that the alert notification is received successfully.

## Outcome
Successfully configured Azure Monitor and Alert Rules to monitor Virtual Machine performance and receive notifications when predefined conditions were met.
