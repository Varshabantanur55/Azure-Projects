# Azure Entra ID - User & Group Management with Role-Based Access Control (RBAC)

## Objective
Manage users and groups in Azure Entra ID and implement Role-Based Access Control (RBAC) to securely manage access to Azure resources based on assigned roles.

## Services Used
- Azure Entra ID
- Azure Role-Based Access Control (RBAC)
- Azure Resource Group
- Azure Portal

## Prerequisites
- Active Azure Subscription
- Azure Portal Access
- Basic understanding of Azure Identity and Access Management

## Steps
1. Create a Resource Group.
2. Open Azure Entra ID in the Azure Portal.
3. Create a new user account.
4. Create a new security group.
5. Add the user to the security group.
6. Verify the user is successfully added to the group.
7. Open the Resource Group and navigate to Access Control (IAM).
8. Assign the *Contributor* role to the security group.
9. Sign in as the assigned user and verify access to the Resource Group.
10. Confirm that the assigned RBAC permissions allow only the authorized actions.

## Outcome
Successfully managed users and security groups in Azure Entra ID, implemented Role-Based Access Control (RBAC), and verified secure access to Azure resources based on assigned roles.
